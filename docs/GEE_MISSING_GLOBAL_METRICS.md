## Notes And Caveats

- Some products are static or epoch-based (e.g., GHS_BUILT_H, Hansen gain, flood hazard maps). For these, keep monthly UI compatibility but treat values as static for a selected epoch/year.
- Coverage/latency caveats from source products matter: e.g., SRTM latitude limits, GRACE coarse footprint, and product-specific validity regions/QA thresholds (especially S5P and MODIS QA bands).
- If you want this reduced to a first implementation sprint, prioritize all `P0` rows first.
